需求文档：一见视觉AI平台
告警推送与内网闭环处理系统

版本：V1.0 | 日期：2025-12-03

1. 项目背景

目前“一见”视觉大模型平台产生的告警仅在原系统内展示。为提高响应效率，需开发一套对接系统，实现以下目标：

  1. 消息推送：将告警实时推送至企业微信和邮件。
  2. 闭环处理：提供内网Web落地页，复刻原平台的详情展示（包含图像/视频的算法图层叠加渲染），并支持“处理/误报”操作。
  3. 数据同步：处理结果实时回写至原平台。

2. 业务流程

  1. 平台回调：一见平台检测到异常，通过 webhook 推送 JSON 数据至我方服务器。
  2. 渲染推送：我方服务器解析数据，生成带有内网落地页链接的消息，发送给相关人员。
  3. 详情查看：用户点击链接进入内网落地页，页面自动加载原始图片/视频，并根据坐标数据动态绘制告警框、电子围栏和绊线。
  4. 处置反馈：用户点击“处理”或“误报”，系统调用一见 API 更新状态。

3. 核心功能模块

模块一：内网告警处置落地页 (Web端)

这是本项目的核心开发工作。需开发一个 Web 页面，通过 URL 参数（如 alarmID）加载告警详情。

3.1 媒体展示区（重难点：图层叠加渲染）

需求描述：页面需使用 HTML5 Canvas 或 SVG 技术，在原始媒体流上方覆盖一层绘图层，用于标识告警触发原因。

A. 图片模式（默认展示）：

    [cite_start]
  • 底层：展示告警抓拍原图（字段：imageURL） [cite: 477]。
    • [cite_start]
  • 叠加层 1（告警目标）：根据 API 返回的 bbox 坐标绘制红色矩形框，标识违规人员/物体 [cite: 764]。
    • [cite_start]
  • 叠加层 2（规则区域）：根据 API 返回的 rois 坐标绘制电子围栏（多边形）或绊线（线段），通常显示为绿色或蓝色 [cite: 759]。

B. 视频模式（点击播放）：

    [cite_start]
  • 底层：播放告警录像片段（字段：videoURL） [cite: 477]。
  • 叠加层（规则区域）：在视频上方持续显示电子围栏或绊线（静态绘制）。注：视频模式下仅需渲染静态的围栏/绊线，无需渲染动态的人体追踪框。

3.2 详细信息区

页面右侧需展示以下关键字段（数据源自 GET /v1/alarms/{id}）：

  • 告警名称：name
    • [cite_start]
  • 发生时间：alarmTime [cite: 477]
    • [cite_start]
  • 设备名称：device.name [cite: 477]
  • 所属部门：deptName
    • [cite_start]
  • 当前状态：status (pending: 待处理, processed: 已处理) [cite: 750]

3.3 处置操作区

  • 若状态为 pending：显示【处理（有效）】和【误报（无效）】两个按钮。
  • 若状态为 processed/mistaken：显示“已处置”标签，禁用按钮。
  • 交互逻辑：点击按钮后，调用一见平台 API 回写状态，成功后刷新页面状态。
