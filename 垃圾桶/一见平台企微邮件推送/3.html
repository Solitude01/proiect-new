需求文档：一见视觉AI平台告警推送与内网闭环处理系统

版本：V1.1（增加高级渲染需求）
日期：2025-12-03

1. 项目背景

目前“一见”视觉大模型平台产生的告警仅在原系统内展示。为了提高响应速度，需开发一套对接系统，将告警信息实时推送到企业微信和邮件，并通过内网独立落地页实现告警详情查看与状态处置。本次更新重点增加了对告警图片和视频的可视化叠加渲染需求（红框、围栏、绊线），以提升点检效率。

2. 业务流程图

  1. 告警产生：一见平台触发告警。
  2. 回调推送：一见平台通过回调接口将数据推送给内部IT服务器。
  3. 消息分发：内部IT服务器解析数据，发送消息至企业微信/邮件（包含带参数的详情页链接）。
  4. 用户点击：用户点击链接，进入“内网告警处置页”。
  5. 高级渲染展示：页面根据坐标数据，在图片/视频上动态绘制告警框及电子围栏/绊线。
  6. 处置与同步：用户点击“处理”或“误报”，内部IT服务器调用API同步状态至一见平台。

模块二：内网告警处置落地页 (Web端)

页面布局参考：参考UI截图，分为左侧媒体区、右侧信息与操作区。

1. 媒体展示区（含高级渲染）[新增重点]

前端需使用 Canvas 或 SVG 技术，在原始媒体层上方叠加绘制分析图层。

媒体类型 渲染需求 数据来源 (API字段)
告警图片 1. 告警红框：绘制红色矩形框，标出违规目标（如人体）。
2. 电子围栏 (ROI)：绘制半透明多边形区域（如黄色填充）。
3. 绊线 (TripWire)：绘制检测线段及方向箭头。 		1. [cite_start]红框：annotations[].bbox [x, y, w, h] [cite: 763-764]
2. [cite_start]围栏/绊线：skill.rois[].points (坐标点集合) 及 kind (类型) [cite: 759, 804-814]
事件视频 1. 静态图层叠加：在视频播放器上层覆盖透明 Canvas。
2. 围栏/绊线：绘制电子围栏和绊线，保持位置与视频画面对齐。
*注：视频中无需动态渲染每帧的AI红框，仅需静态显示规则区域（围栏/线）。 		同上，使用 skill.rois 数据绘制规则区域。
技术提示： API返回的坐标通常是像素坐标或归一化坐标，前端渲染时需根据图片/视频的实际分辨率进行比例换算，确保图层位置准确。

3. 处置操作区（闭环核心）

  • 状态判断：
    • 若 status 为 pending：显示“处理（有效）”和“误报（无效）”按钮。
    • 若 status 为 processed：隐藏按钮，仅显示“已处置”状态标签。
  • 操作逻辑：点击按钮后调用后台接口，同步状态至一见平台，成功后刷新页面状态。

4. 技术接口映射 (API Reference)

请开发团队依据《一见-视觉大模型平台API文档（V4.6.0）》开发。

Step 1: 获取渲染数据 (告警详情)

    [cite_start]
  • 接口：GET /v1/alarms/{alarmID} [cite: 468]
  • 关键渲染数据解析：
    • imageURL: 原始告警底图。
    • videoURL: 原始告警视频。
    • annotations: 目标检测框数组。例如：bbox: [140.375, 230.875, 26.875, 66.375] (x, y, w, h)。
    • skill.rois: 规则区域数组。
      • points: 坐标点列表 [x1, y1, x2, y2...]。
      • kind: roi (多边形) 或 tripWire (绊线)。

Step 2: 状态回写 (处理/误报)

    [cite_start]
  • 接口：PUT /v1/alarms/{uuid} [cite: 724]
  • 参数 Body：
    • 点击“处理”发送：{ "processType": "efficient" }
    • 点击“误报”发送：{ "processType": "invalid" }

5. 非功能性需求

  • 内网穿透：平台返回的媒体 URL 若为内网 IP，需在 IT 服务器做反向代理，确保企业微信（外网）可访问图片。
  • 数据安全：内网落地页需验证用户身份，防止未授权访问。
  • 兼容性：落地页需适配 PC 浏览器及企业微信内置浏览器。
