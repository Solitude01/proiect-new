# 反向代理 & 内网穿透 详细实施指南

针对你的 K8s + Istio 架构 (`http://ds.scc.com.cn/v1/chat/completions`)

---

## 方案一：反向代理 (Nginx/Caddy)

### 架构图
```
用户 → 公网服务器(nginx/caddy) → 内网穿透/VPN → K8s Ingress → Istio → Pod
```

### 1.1 Nginx 反向代理

#### 部署步骤

**公网服务器安装 Nginx**
```bash
# Ubuntu/Debian
sudo apt update && sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx
```

**配置文件**
```nginx
# /etc/nginx/sites-available/model-api
server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    # SSL 证书
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    # 安全增强
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256';
    ssl_prefer_server_ciphers off;

    # 限制请求频率
    limit_req_zone $binary_remote_addr zone=api_limit:10m rate=10r/s;
    limit_req zone=api_limit burst=20 nodelay;

    location /v1/chat/completions {
        proxy_pass http://ds.scc.com.cn/v1/chat/completions;
        proxy_http_version 1.1;

        # 关键头部
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # API 认证透传
        proxy_set_header Authorization $http_authorization;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 300s;  # 大模型响应慢，需要长超时

        # 流式响应支持
        proxy_buffering off;
        proxy_cache off;
    }
}

# HTTP 自动跳转 HTTPS
server {
    listen 80;
    server_name api.yourdomain.com;
    return 301 https://$server_name$request_uri;
}
```

**启用配置**
```bash
sudo ln -s /etc/nginx/sites-available/model-api /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### Nginx 方案优缺点
- ✅ 性能极佳，成熟稳定
- ✅ 社区资源丰富
- ❌ 配置较繁琐，证书需手动管理

### 1.2 Caddy 反向代理 ⭐更简单

#### 部署步骤

**安装 Caddy**
```bash
# 一键安装
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy
```

**配置文件** (Caddyfile)
```caddyfile
# /etc/caddy/Caddyfile
api.yourdomain.com {
    # 自动 HTTPS (Let's Encrypt)
    tls your-email@example.com

    # 反向代理到内网接口
    reverse_proxy ds.scc.com.cn {
        # 健康检查
        health_uri /health
        health_interval 30s

        # 超时设置
        transport http {
            dial_timeout 30s
            response_header_timeout 300s
        }

        # 请求头透传
        header_up Host {upstream_hostport}
        header_up X-Real-IP {remote}
        header_up X-Forwarded-For {remote}
        header_up X-Forwarded-Proto {scheme}
    }

    # 速率限制 (需安装 caddy-rate-limit 插件)
    # rate_limit {
    #     zone static_api {
    #         key static
    #         events 100
    #         window 1m
    #     }
    # }

    # 日志
    log {
        output file /var/log/caddy/api-access.log
        format json
    }
}
```

**重载配置**
```bash
sudo systemctl reload caddy
```

#### Caddy 方案优缺点
- ✅ 配置极简，自动 HTTPS
- ✅ 现代化设计，HTTP/3 支持
- ❌ 插件生态不如 Nginx 丰富

## 方案二：内网穿透 (frp)

### 架构图
```
用户 → 公网服务器(frps) ←──隧道── frpc(内网) → K8s Ingress → Istio → Pod
```

### 2.1 frp 快速部署

#### 服务端配置 (公网服务器)

**下载安装**
```bash
# 下载最新版
wget https://github.com/fatedier/frp/releases/download/v0.60.0/frp_0.60.0_linux_amd64.tar.gz
tar -xzf frp_0.60.0_linux_amd64.tar.gz
cd frp_0.60.0_linux_amd64

# 移动二进制文件
sudo cp frps /usr/local/bin/
sudo mkdir -p /etc/frp
```

**服务端配置** `/etc/frp/frps.toml`
```toml
# 监听端口
bindPort = 7000

# 可选：Web 控制台
webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "your-strong-password"

# 身份验证
token = "your-secure-token-here"

# 多路复用
tcpmuxHTTPConnectPort = 5002

# 日志
log.to = "/var/log/frps.log"
log.level = "info"
log.maxDays = 30
```

**Systemd 服务** `/etc/systemd/system/frps.service`
```ini
[Unit]
Description=FRP Server
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/frps -c /etc/frp/frps.toml
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

**启动服务**
```bash
sudo systemctl daemon-reload
sudo systemctl enable frps
sudo systemctl start frps
```

#### 客户端配置 (内网机器)

**客户端配置** `/etc/frp/frpc.toml`
```toml
# 服务端地址
serverAddr = "your-public-server-ip"
serverPort = 7000

# 身份验证
token = "your-secure-token-here"

# 日志
log.to = "/var/log/frpc.log"
log.level = "info"
log.maxDays = 30

# 连接池
transport.poolCount = 10

# 代理配置
[[proxies]]
name = "model-api-http"
type = "http"
localIP = "ds.scc.com.cn"
localPort = 80
customDomains = ["model.yourdomain.com"]

# TCP 代理（如果 HTTP 不行）
[[proxies]]
name = "model-api-tcp"
type = "tcp"
localIP = "ds.scc.com.cn"
localPort = 80
remotePort = 8080
```

**启动客户端**
```bash
frpc -c /etc/frp/frpc.toml
```

### 2.2 frp + Nginx 组合 (推荐)

在公网服务器上，frp 接收流量后，通过 Nginx 提供 HTTPS 和高级功能：

```
用户 → Nginx(443) → frps(7000) → frpc → ds.scc.com.cn
```

**Nginx 配置**
```nginx
server {
    listen 443 ssl http2;
    server_name model.yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;  # frp 映射的本地端口
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        proxy_buffering off;
        proxy_read_timeout 300s;
    }
}
```

### 2.3 frp 方案优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 无需公网 IP | ❌ 依赖中转服务器 |
| ✅ 部署简单，5分钟搞定 | ❌ 性能受限于中转带宽 |
| ✅ 开源免费 | ❌ 稳定性不如直连 |
| ✅ 支持 TCP/UDP/HTTP/HTTPS | ❌ 大模型长响应可能超时 |
| ✅ 自带 Web 控制台 | ❌ 需要维护额外组件 |

## 方案对比总结

| 维度 | Nginx 反向代理 | Caddy 反向代理 | frp 内网穿透 |
|------|----------------|----------------|--------------|
| **部署难度** | 中 | 低 | 低 |
| **配置复杂度** | 高 | 低 | 中 |
| **HTTPS 配置** | 手动 | 自动 | 需配合 Nginx |
| **性能** | 极高 | 高 | 中等 |
| **稳定性** | 极高 | 高 | 中等 |
| **成本** | 需公网服务器 | 需公网服务器 | 需公网服务器 |
| **适用场景** | 生产环境 | 中小项目 | 开发测试/无公网IP |

---

## 快速选择建议

```
有公网服务器 + 生产环境    →  Caddy (简单) / Nginx (性能)
有公网服务器 + 开发测试    →  frp + Nginx/Caddy
无公网服务器              →  frp (自建服务端) / ngrok(付费) / Cloudflare Tunnel
```

## 安全加固建议

### 1. 访问控制
```nginx
# IP 白名单
allow 192.168.1.0/24;
allow 10.0.0.0/8;
deny all;

# API Key 验证
if ($http_authorization != "Bearer your-secret-key") {
    return 401;
}
```

### 2. 速率限制
```nginx
limit_req_zone $binary_remote_addr zone=api:10m rate=5r/s;
limit_req zone=api burst=10 nodelay;
```

### 3. 连接安全
```toml
# frp 启用 TLS
transport.tls.enable = true
transport.tls.certFile = "/path/to/cert.pem"
transport.tls.keyFile = "/path/to/key.pem"
```

## 一键部署脚本

### frp 服务端一键安装
```bash
#!/bin/bash
# install_frps.sh

FRP_VERSION="0.60.0"
wget https://github.com/fatedier/frp/releases/download/v${FRP_VERSION}/frp_${FRP_VERSION}_linux_amd64.tar.gz
tar -xzf frp_${FRP_VERSION}_linux_amd64.tar.gz
cd frp_${FRP_VERSION}_linux_amd64
sudo cp frps /usr/local/bin/

# 配置
sudo mkdir -p /etc/frp
sudo tee /etc/frp/frps.toml << 'EOF'
bindPort = 7000
token = "$(openssl rand -base64 32)"
EOF

# 服务
sudo tee /etc/systemd/system/frps.service << 'EOF'
[Unit]
Description=FRP Server
After=network.target
[Service]
Type=simple
ExecStart=/usr/local/bin/frps -c /etc/frp/frps.toml
Restart=always
[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable frps
sudo systemctl start frps

echo "FRP Server installed. Token: $(grep token /etc/frp/frps.toml | cut -d'"' -f2)"
```

运行：`chmod +x install_frps.sh && sudo ./install_frps.sh`
