一句话总结

构建了一个**“AI 演讲稿流水线工厂”**：用户只需投喂一个 PPT，系统就会自动调度“分析师”、“导演”、“作家”、“质检员”和“编辑”协同工作，最终在网页上把写好的稿子即时推送到用户面前。


1. 核心定义

这是一个基于 n8n 编排与 LLM（大模型）驱动的自动化流水线。它能将用户上传的 PPT 文件，通过智能分析、策划、撰写、润色，最终转化为适合数字人口播的、带有情绪和节奏标记的逐页演讲稿，并实时向前端反馈处理进度。
2. 系统架构与数据流向

整个系统由三部分组成，形成了一个闭环：

    前端 (Web UI):

        负责文件上传（.pptx）。

        基于 task_id 轮询后端接口，实时展示进度（从“已上传”到“大纲生成”再到“逐页脚本完成”）。

    中台 (Python/FastAPI + SQLite):

        作为任务调度中心和数据仓库。

        接收文件并存储，触发 n8n Webhook。

        提供 /api/update_status 接口供 n8n 回调，将复杂的异步处理状态持久化到 SQLite 数据库中。

    后台引擎 (n8n Workflow):

        核心业务逻辑的执行者，包含复杂的 AI Agent 链条。

3. 核心功能流程（AI 处理链）

 n8n 工作流设计模拟了真实的人类专业团队分工：

    解析与提炼 (Extraction):

        利用 PowerPoint to Text 节点提取 PPT 原始内容。

        AI 助教：快速提炼每一页的核心主题和关键要点。

        聚合大纲：将散落的页面信息聚合成一份完整的课程大纲。

    顶层策划 (Director Agent):

        AI 总导演：拥有全局视角，分析大纲逻辑，为每一页分配“情绪基调（Mood）”和“转场指令”，确保演讲的起承转合自然流畅。

    多智能体精细化写作 (Writer-Critic-Editor Loop):

        针对每一页 PPT，执行“三步走”策略：

            Writer（初稿）：根据 PPT 内容和导演指令，生成初稿。

            Critic（毒舌教官）：这是亮点。不直接修改，而是提出“口语化”、“呼吸感”、“对象感”方面的尖锐批评。

            Editor（金牌精修）：根据批评意见重写脚本，标注停顿，确保产出的是“人话”而不是“书面语”。

    实时状态同步 (Real-time Feedback):

        在“分页完成”、“大纲完成”、“策划完成”、“单页生成”、“全部完成”等关键节点，通过 HTTP Request 回调 Python 后端。

        采用 动态 Key (slide_N_script) 策略，确保每一页生成的脚本都能独立保存到数据库，避免数据覆盖。

4. 技术亮点

    健壮的容错机制: 在 n8n 中实现了对 task_id 的多重获取策略（优先 Body，次选 Query，兜底 Error），解决了 HTTP 请求参数位置不一致导致流程中断的问题。

    异步与解耦: 前端不直接等待 AI 生成（耗时久），而是通过“上传 -> 轮询”机制，配合 n8n 的异步回调，体验极佳。

    数据结构优化: 解决了 SQLite UPDATE 操作覆盖旧数据的问题，利用 JSON 结构的灵活性，实现了增量式的脚本存储。

    高质量的内容生成: 引入了 Reflection（反思）机制（即 Critic 节点），比单纯的一次性生成（Zero-shot）能产出质量高得多的演讲稿。
