## 部署前准备

### 1. 服务器环境要求

```bash
# 安装Docker (CentOS/RHEL)
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io
sudo systemctl start docker
sudo systemctl enable docker

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 2. 网络连通性检查

```bash
# 检查到数据库的连通性
ping 10.30.5.92
ping 10.20.32.165

# 检查数据库端口
telnet 10.30.5.92 3306
telnet 10.20.32.165 3306

# 检查到互联网的连通性（用于Gemini API）
curl https://generativelanguage.googleapis.com
```

### 3. 防火墙配置

```bash
# 开放必要端口
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

## 故障排除

### 问题1: 容器启动失败

```bash
# 查看详细日志
docker-compose logs web

# 检查配置文件
docker-compose config

# 检查端口占用
netstat -tulpn | grep :8000
```

### 问题2: 数据库连接失败

```bash
# 检查网络连通性
docker exec ai_capability_web ping 10.30.5.92

# 检查数据库配置
docker exec ai_capability_web cat .env

# 测试数据库连接
docker exec ai_capability_web python -c "from app.database import db_manager; db_manager.get_connection('nantong')"
```

### 问题3: Nginx代理失败

```bash
# 检查Nginx配置
docker exec ai_capability_nginx nginx -t

# 重新加载Nginx配置
docker exec ai_capability_nginx nginx -s reload
```
