## 2. 建表步骤

### 2.1 创建数据库（如果不存在）
```sql
CREATE DATABASE IF NOT EXISTS ai_project_data 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 2.2 执行建表脚本

**方式一：命令行执行**
```bash
# 南通数据库
mysql -h 10.30.5.92 -u ai_project_data_read -pai_project_data_read ai_project_data < create_tables.sql

# 无锡数据库
```

## 3. 表结构说明

### 3.1 ai_projects - 项目主表
存储AI项目的所有数据，包括基础信息和AI分析结果。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键ID |
| project_name | VARCHAR(255) | 项目名称 |
| factory_name | VARCHAR(100) | 工厂名称 |
| project_goal | TEXT | 项目目标 |
| benefit_desc | TEXT | 收益描述 |
| ok_image_desc | TEXT | OK图片描述 |
| ng_image_desc | TEXT | NG图片描述 |
| application_scenario | TEXT | 应用场景简述 |
| processing_object | TEXT | 处理对象(输入) |
| core_functions | JSON | 核心功能（数组） |
| output_interface | TEXT | 输出形式/接口 |
| deployment_method | TEXT | 部署方式 |
| sync_status | ENUM | 同步状态 |
| source_region | VARCHAR(50) | 数据来源区域 |

### 3.2 sync_tasks - 同步任务表
记录所有数据同步任务的执行状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 任务ID |
| task_type | ENUM | 任务类型 |
| status | ENUM | 任务状态 |
| total_count | INT | 总数量 |
| processed_count | INT | 已处理数量 |
| error_message | TEXT | 错误信息 |
| source_file | VARCHAR(500) | 源文件路径 |

### 3.3 sync_logs - 同步日志表
记录同步任务的详细日志。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 日志ID |
| task_id | INT | 关联任务ID |
| log_level | ENUM | 日志级别 |
| message | TEXT | 日志消息 |

### 3.4 factories - 工厂信息表
工厂基础信息，用于筛选和统计。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 工厂ID |
| factory_code | VARCHAR(50) | 工厂编码 |
| factory_name | VARCHAR(100) | 工厂名称 |
| region | VARCHAR(50) | 所属区域 |

## 6. 常见问题

### Q1: 连接超时
检查服务器防火墙设置，确保3306端口开放。

### Q2: 权限不足
确认数据库用户有CREATE TABLE权限。

### Q3: 字符集问题
确保数据库使用utf8mb4字符集，避免中文乱码。
