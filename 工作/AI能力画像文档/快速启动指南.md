# AI能力画像文档系统 - 快速启动指南

## 1. 环境要求

- Docker 20.10+
- Docker Compose v2.0+
- Python 3.9+（本地开发）

## 2. 快速启动

### 方式一：Docker Compose（推荐）

```bash
# 1. 克隆代码
cd D:\proiect\工作\AI能力画像文档

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置数据库连接和API密钥

# 3. 启动所有服务
docker-compose up -d

# 4. 查看服务状态
docker-compose ps

# 5. 查看日志
docker-compose logs -f web
docker-compose logs -f worker
```

启动后访问：
- Web应用：http://localhost
- API文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

### 方式二：本地开发

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件

# 3. 初始化数据库
python test_db_connection.py

# 4. 启动应用
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

访问 http://localhost:8000/docs 查看API文档

## 3. 数据库初始化

### 3.1 创建数据库
```sql
CREATE DATABASE IF NOT EXISTS ai_project_data 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行建表脚本
```bash
# 南通数据库
mysql -h 10.30.5.92 -u ai_project_data_read -pai_project_data_read ai_project_data < create_tables.sql

# 无锡数据库
mysql -h 10.20.32.165 -u ai_project_data_read -pai_project_data_read ai_project_data < create_tables.sql
```

## 4. 主要API端点

### 项目管理
- `GET /api/projects` - 获取项目列表
- `GET /api/projects/{id}` - 获取项目详情
- `POST /api/projects` - 创建项目
- `PUT /api/projects/{id}` - 更新项目
- `DELETE /api/projects/{id}` - 删除项目

### 数据同步
- `POST /api/sync/trigger?task_type=excel_import&source_file=/path/to/file.xlsx` - 触发同步
- `POST /api/sync/upload` - 上传Excel文件
- `GET /api/sync/tasks` - 获取同步任务列表
- `GET /api/sync/status` - 获取同步状态

### AI分析
- `POST /api/analyze/batch` - 批量AI分析
- `POST /api/analyze/{project_id}` - 单个项目分析
- `GET /api/analyze/status/{task_id}` - 获取分析状态

## 7. 常见问题

### Q1: 数据库连接失败
检查 `.env` 文件中的数据库配置是否正确，确保服务器能访问数据库主机。

### Q2: 端口冲突
如果8000端口被占用，修改 `docker-compose.yml` 中的端口映射：
```yaml
ports:
  - "8001:8000"  # 改为8001
```

### Q3: 文件上传失败
检查 `nginx.conf` 中的 `client_max_body_size` 设置，默认50MB。

### Q4: AI分析不工作
确保配置了 `GEMINI_API_KEY`，或者使用模拟数据模式（不配置API密钥时使用）。
