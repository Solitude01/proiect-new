## 一、辅助监控系统（新增需求）
### 辅助监控 —— LA 预警对接

1. **预警对接方式**
   - LLA 通过 HTTP 请求（Webhook）发送预警信息，请求体为 JSON 格式。
   - 预警内容包含以下字段：

   | 字段 | 类型 | 说明 | 示例 |
   |------|------|------|------|
   | event_id | String | 事件唯一标识（项目管理中配置） | "电感测试1的唯一识别ID" |
   | event_time | String | 预警触发时间 | "2026-02-10 13:32:16.568" |
   | beginTime | String | 回放视频开始时刻 | "2026-02-10 13:34:05.552" |
   | endTime | String | 回放视频结束时刻 | "2026-02-10 13:35:05.568" |
   | Deepmind | String | 超脑编号，用于查找对应 NVR | "20" |
   | Channel | String | NVR 通道号 | "2" |   `基于辅助监控系统优化需求260120 字段优化过后.pdf` 
   | Task Result | String | 检测结果（OK/NG） | "NG" |

2. **NVR 设备查找**(可复用 辅助监控系统-超脑配置的内容)
   - 系统收到预警后，根据 `Deepmind` 编号从配置表（`Deepmind.json`）中查找 NVR 的连接信息。
   - 配置表结构（每条记录）：
     ```json
     {
       "Deepmind": "20",       // 超脑编号，与预警数据中的 Deepmind 字段对应
       "IP": "10.30.5.112",    // NVR 设备 IP
       "Password": "wdzn6688", // NVR 登录密码
       "HttpPort": 80,         // HTTP 端口（默认 80）
       "RtspPort": 554,        // RTSP 端口（默认 554）
       "Scheme": "http"        // 协议（http/https）
     }
     ```
   - 用户名统一为 `admin`。
   - 当前已配置 Deepmind 10~20 共 11 台 NVR。

3. **视频回放下载流程**
   ```
   1. 接收 HTTP 预警请求（JSON body）
   2. 解析 Deepmind 编号 → 查 Deepmind.json → 得到 NVR 连接信息
   3. 时间格式转换："2026-02-10 13:34:05.552" → "20260210T133405Z"
   4. 拼接海康 ISAPI 下载 URL：
      {scheme}://{IP}:{HttpPort}/ISAPI/ContentMgmt/download?playbackURI=rtsp://{IP}:{RtspPort}/Streaming/tracks/{Channel}01?starttime={begin}%26endtime={end}
   5. 使用 HTTP Digest Auth 认证，流式下载 MP4 视频
   6. 保存到指定目录：/{base_dir}/{event_id}/ch{Channel}_{begin}_{end}.mp4
   ```

4. **要点**
   - 认证方式：HTTP Digest Auth (MD5)
   - playbackURI 中的 `&` 必须编码为 `%26`
   - 通道号规则：Channel + "01" = trackID（如通道 5 → 501，通道 11 → 1101）
   - 01 为主码流，02 为子码流
   - 时间精度：毫秒部分被丢弃，精确到秒

5. **项目管理与任务点检**
   - 在项目管理中为每个项目配置唯一 ID。
   - 平台基于 HTTP 请求向超脑获取准确的视频数据。
   - 视频结果需在项目任务中展示，用于点检查看。
   - 项目增加配置项-配置起始偏移（preSeconds）和结束偏移（postSeconds）、尾部截取长度（tailSeconds）。
   - 时间窗口 beginTime2 和 endTime2 决定时间段范围 ，

6. **参考实现**
   - 正式脚本：`download_event.py`
   - NVR 配置表：`Deepmind.json`
   - 接口文档：`接口文档.md`
