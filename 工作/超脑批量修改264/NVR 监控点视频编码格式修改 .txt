第一性原理拆解

修改 NVR 监控点视频编码格式的本质，是通过 RESTful 架构的 ISAPI 接口，向目标设备的特定通道（例如通道 101）提交包含期望参数的 XML 状态描述文件。设备接收到请求后，会解析该 XML 并直接更改底层的音视频编码器配置。


修改方案

通过审查你提供的请求抓包数据，核心修改点明确位于 XML 荷载的 <Video> 模块内。


1. 定位目标接口与关键参数

要将编码格式修改为 H.264，需要向指定的接口发送 PUT 请求 ，并指定对应的编码类型：


 * 目标接口 (URL): http://10.30.5.112/ISAPI/ContentMgmt/StreamingProxy/channels/101
   

 * 关键 XML 节点: StreamingChannel -> Video -> videoCodecType
   

在捕获的荷载中，该节点实际上已经设定为 H.264 。要执行这项修改，只需要提取这部分 XML，并确保 <videoCodecType> 的值为 H.264（若要改回 H.265，对应修改为 H.265 即可）。



2. 完整荷载结构

3. 自动化集成思路

在编写 Python 脚本与 NVR 的 ISAPI 接口进行自动化交互或视频录像下载集成时，可以直接利用 requests 库下发 PUT 请求。需要特别注意处理好设备的认证机制：

Python


import requests
from requests.auth import HTTPDigestAuth

url = "http://10.30.5.112/ISAPI/ContentMgmt/StreamingProxy/channels/101"
# payload 为上述包含 <videoCodecType>H.264</videoCodecType> 的完整 XML 字符串
payload = """<?xml version="1.0" encoding="UTF-8"?><StreamingChannel>...</StreamingChannel>""" 

headers = {
    'Content-Type': 'application/xml; charset="UTF-8"'
}

# ISAPI 接口通常采用 Digest 认证而非 Basic 认证
response = requests.put(
    url, 
    data=payload.encode('utf-8'), 
    headers=headers, 
    auth=HTTPDigestAuth('admin', 'your_password')
)

print(response.status_code)



批判性评估

 * 优势：利用 ISAPI 接口进行 PUT 操作，能够脱离 Web 界面实现设备配置的标准化与自动化下发。这种方式响应速度快，且非常适合集成到智能制造或视频 AI 监控回调的业务流中。

 * 劣势（风险）：
   
   1. 参数覆盖风险：PUT 请求在 RESTful 语义中通常代表全量覆盖。如果在自行构造的 XML 中漏掉了分辨率、帧率（如 maxFrameRate）或码率上限（如 vbrUpperCap ）等参数，NVR
      设备有极大概率会将这些缺失项重置为出厂默认值。最佳实践是先通过 GET 请求拉取当前配置，仅在原文本上做正则或 DOM 树替换后再 PUT 回去。
      
   
   2. 资源消耗变化：将编码由 H.265 降级为 H.264，虽然极大提升了通过 ISAPI 下载录像片段后在常规播放器及 AI 目标检测模型（如 YOLO 系列）中的解码兼容性，但其劣势在于会显著增加 NVR 的存储开销与网络传输带宽占用。

2. 自动化 PYTHON 脚本

这个脚本会自动处理 Deepmind.json 中的所有设备，先校验通道，再检查编码格式，最后仅对非 H.264 的通道进行修改。

Python


import json
import requests
from requests.auth import HTTPDigestAuth
import xml.etree.ElementTree as ET

# 1. 加载设备列表
with open('Deepmind.json', 'r', encoding='utf-8') as f:
    devices = json.load(f)

def get_online_channels(ip, port, auth):
    """步骤1：校验该超脑有哪些监控点"""
    url = f"http://{ip}:{port}/ISAPI/ContentMgmt/InputProxy/channels"
    try:
        response = requests.get(url, auth=auth, timeout=5)
        if response.status_code == 200:
            root = ET.fromstring(response.text)
            # 提取所有 InputProxyChannel 的 ID
            ns = {'ns': 'http://www.isapi.org/ver20/XMLSchema'}
            # 注意：XML命名空间可能变化，这里做简单处理
            channels = [child.find('{*}id').text for child in root.findall('.//{*}InputProxyChannel')]
            return channels
        else:
            print(f"[{ip}] 获取通道列表失败: {response.status_code}")
            return []
    except Exception as e:
        print(f"[{ip}] 连接异常: {e}")
        return []

def modify_codec(ip, port, auth, channel_id):
    """步骤2：读取并修改指定通道的编码格式"""
    # 拼接主码流ID，例如通道1 -> 101
    stream_id = f"{channel_id}01" 
    
    # 这里的URL依据是你提供的 1.txt (NVR代理通道配置)
    url = f"http://{ip}:{port}/ISAPI/ContentMgmt/StreamingProxy/channels/{stream_id}"
    
    try:
        # A. 获取当前配置 (GET)
        resp_get = requests.get(url, auth=auth, timeout=5)
        if resp_get.status_code != 200:
            print(f"  - [{stream_id}] 获取配置失败")
            return

        # B. 解析XML并修改
        xml_content = resp_get.text
        root = ET.fromstring(xml_content)
        
        # 查找 videoCodecType 节点 (忽略命名空间查找)
        codec_node = root.find('.//{*}videoCodecType')
        
        if codec_node is not None:
            current_codec = codec_node.text
            if current_codec == 'H.264':
                print(f"  - [{stream_id}] 已经是 H.264，跳过")
                return
            
            # 修改为 H.264
            print(f"  - [{stream_id}] 当前为 {current_codec}，正在修改为 H.264...")
            codec_node.text = 'H.264'
            
            # C. 回写配置 (PUT)
            headers = {'Content-Type': 'application/xml'}
            new_xml = ET.tostring(root, encoding='utf-8', method='xml')
            resp_put = requests.put(url, data=new_xml, headers=headers, auth=auth)
            
            if resp_put.status_code == 200:
                print(f"  - [{stream_id}] 修改成功！")
            else:
                print(f"  - [{stream_id}] 修改失败: {resp_put.status_code} {resp_put.text}")
        else:
            print(f"  - [{stream_id}] XML中未找到编码节点")

    except Exception as e:
        print(f"  - [{stream_id}] 操作异常: {e}")

# 主循环
print("开始执行批量检查与修改...")
for dev in devices:
    ip = dev['IP']
    port = dev['HttpPort']
    password = dev['Password'] # 从json中根据设备读取对应密码
    
    print(f"\n正在连接设备: {ip} (ID: {dev.get('Deepmind', '?')})")
    
    auth = HTTPDigestAuth('admin', password)
    
    # 1. 先校验有哪些通道
    channels = get_online_channels(ip, port, auth)
    
    if channels:
        print(f"  发现通道: {channels}")
        # 2. 逐个修改
        for ch in channels:
            modify_codec(ip, port, auth, ch)
    else:
        print("  未发现有效通道或连接失败")
