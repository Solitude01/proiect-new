## 一、核心功能需求

### 模式一：单视频切分
- 用户选择单个视频文件
- 默认输出路径：在该视频文件所在目录下自动创建「切分图片」文件夹
- 支持自定义输出路径（可手动输入或点击选择目录按钮覆盖默认路径）

### 模式二：单目录切分
- 用户选择一个目录
- 对该目录下**直接包含的**所有视频文件（不递归子目录）作为整体进行切分
- 默认输出路径：在所选目录下自动创建「切分图片」文件夹
- 支持自定义输出路径

### 模式三：多目录并发切分
- 用户可选择**多个**目录（支持逐个添加、也支持批量添加）
- 各目录下的视频并发处理（各目录任务互相独立）
- 每个目录默认在其自身路径下创建「切分图片」子文件夹输出结果
- 支持为**每个目录单独指定**自定义输出路径
- 界面以列表/卡片形式展示各目录条目，每条目显示：目录路径、输出路径（可编辑）、当前进度状态

## 二、UI/UX 交互流程

**关键原则：模式选择必须在进入主操作区前完成，作为第一步明确引导用户。**

### 启动流程设计
1. 工具启动后，**首屏**以清晰、视觉突出的方式呈现三种模式的选择卡片
2. 每张卡片包含：模式名称、图标、一句话描述
3. 用户点击/选中某模式后，界面以平滑过渡动画切换到对应的操作区
4. 操作区顶部保留模式标签，支持切换模式时有确认提示（防止误操作丢失已有配置）

### 交互细节
- 路径输入框支持直接拖拽文件/目录到框内
- 模式三的目录列表支持拖拽排序、单条删除、一键清空
- 所有默认路径在用户未手动修改时以灰色"占位提示"样式展示（区别于已确认的自定义路径）
- 执行前自动校验：视频文件是否存在、输出路径是否可写，错误以内联提示展示

## 三、UI 视觉设计规范

> 使用 tkinter + ttkbootstrap（或 customtkinter）实现现代化视觉风格，禁止使用默认的系统原生控件样式。

### 设计方向：「工业精密感」+ 深色主题
- **色调**：深灰背景（#1a1d23）+ 主强调色冷蓝（#4a9eff）+ 辅助色暗金（#c8a96e），营造专业工具感
- **字体**：标题使用等宽/工程风字体（如 JetBrains Mono / Consolas），正文使用干净的无衬线（如 Segoe UI / PingFang SC）
- **组件风格**：
  - 选择卡片：圆角矩形，悬停时有边框高亮 + 轻微上浮阴影动效
  - 按钮：扁平化设计，带左侧彩色竖线标识主操作
  - 进度条：细线型，带百分比数字，完成时变绿色
  - 列表条目（模式三）：卡片式，左侧有状态色块（等待/运行中/完成/错误）
- **动效**：模式切换时面板淡入淡出（alpha 渐变）；任务完成时条目有短暂"呼吸"高亮

### 布局结构

```
│  ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│  │单视频切分 │ │单目录切分 │ │多目录并发 │     │
│  └──────────┘ └──────────┘ └──────────┘     │
├─────────────────────────────────────────────┤
│  ② 操作区（选模式后展开）                       │

├─────────────────────────────────────────────┤
│  ③ 执行区                                    │

```

## 四、技术实现要求

- 语言：Python 3.8+
- GUI 框架：tkinter + ttkbootstrap（首选）或 customtkinter
- 多目录并发：使用 `concurrent.futures.ThreadPoolExecutor`，线程数可配置
- FFmpeg 调用：复用现有的帧切分逻辑，不改变核心算法
- 日志区：实时滚动显示 FFmpeg 输出和状态信息，支持一键复制全部日志
- 配置持久化：记忆上次使用的模式和路径，下次启动时自动恢复
