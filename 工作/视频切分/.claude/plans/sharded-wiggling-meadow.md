视频帧提取工具紧凑布局重设计计划

## 背景

用户反馈当前UI布局过于松散，需要重新设计为更紧凑的布局，提高信息密度，减少不必要的留白。

## 当前问题分析

1. **垂直间距过大** - 每个卡片内部有过多padding，卡片之间间距也较大
2. **元素间距松散** - 输入框、标签、按钮之间留白过多
3. **标题区域过高** - 深色标题栏占据过多空间（90px+）
4. **卡片内边距过大** - 左右20px、上下15-18px的padding显得松散
5. **整体高度过高** - 默认800x750的尺寸对于紧凑内容来说过大

## 紧凑化设计方向

### 目标
- 减少垂直方向padding（从20px减少到8-12px）
- 减小元素间距（从15px减少到5-8px）
- 降低标题栏高度（从90px减少到60px）
- 减小卡片内边距（从20px减少到12-15px）
- 字体微调，减小行高
- 整体尺寸缩小（默认700x600）

### 具体修改点

**1. 标题栏紧凑化**
- 高度：90px → 60px
- 标题字号：20px → 18px
- 副标题字号：9px → 8px
- 内边距：15px → 10px

**2. 卡片紧凑化**
- 卡片标题内边距：(18,0) → (12,0)
- 分隔线内边距：(20,0) → (12,0)
- 内容区域内边距：20px → 12px
- 卡片间距：15px → 10px

**3. 输入区域紧凑化**
- 输入框高度：ipady=6 → ipady=4
- 标签间距：从多行压缩为单行或紧凑排列
- 按钮尺寸：padding从18px减少到12px
- 快捷按钮间距：6px → 4px

**4. 进度区域紧凑化**
- 状态文本内边距：(15,5) → (10,3)
- 百分比标签内边距：(0,5) → (0,3)
- 当前文件内边距：(0,15) → (0,10)

**5. 底部按钮区域紧凑化**
- 主按钮padding：(40,14) → (30,12)
- 辅助按钮padding：(20,10) → (15,8)
- 辅助按钮间距：8px → 6px

**6. 整体尺寸调整**
- 默认窗口大小：800x750 → 700x600
- 最小尺寸：650x600 → 550x500
