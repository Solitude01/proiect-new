# 1. 项目概述与架构约束
需在本地 Windows 系统部署一套轻量级 Web 服务中间件，作为底层逻辑编排软件（LogicAgent，简称 LA）与现场操作员的交互门户。
**核心突破点**：系统必须支持**多实例运行（Multi-Instance）**。启动一个服务端后，支持创建多个独立的实例（如“一车间实例”、“二车间实例”），每个实例拥有独立的配置、独立的 Web 控制台 URL 和独立的数据隔离通道，供不同产线的人员互相无干扰地使用。

**技术栈强制要求：**
* **后端**：Python 3 + FastAPI + Uvicorn。
* **前端**：HTML5 + 原生 JavaScript + TailwindCSS (CDN 引入)。禁止使用需 npm 编译的框架（如 Vue CLI / React）。
* **数据存储**：采用本地基于 `instance_id` 的 JSON 文件管理（如 `configs/` 目录下存储各个实例的 json），或单一的 `master_config.json` 包含嵌套的实例字典。

---

# 2. 核心系统缺陷规避要求（极高优先级）
必须在代码中完美解决以下 5 个物理/环境限制：
1. **多实例数据串线（致命）**：WebSocket 必须实现房间（Room）机制。LA 推送给 `/api/{instance_id}/webhook` 的数据，只能广播给连接在 `/ws/{instance_id}` 的前端客户端。
2. **前后端实时断层**：LA 只能通过 HTTP POST 推送数据。后端接收后，必须通过 WebSocket 实时推给前端。
3. **浏览器音频沙盒限制**：主流浏览器禁止自动播放音频。前端页面必须设计一个全局的“连接并解锁音频”的交互按钮。
4. **状态黑盒**：前端必须具备 WebSocket 重连机制，且页面顶部必须有明确的**“网络状态指示灯（在线/离线）”**。
5. **Windows 防火墙拦截**：必须提供一个 `.bat` 启动脚本，包含安装依赖与 `netsh advfirewall` 一键放行端口的命令。

# 3. 核心功能模块划分

### 3.1 后端服务 (`main.py`)
* **实例管理器**：提供创建新实例的接口，自动生成实例的配置结构。
* **Webhook 接收器 (`/api/{instance_id}/webhook`)**：接收对应 LA 实例发送的 JSON 数据。
* **隔离 WebSocket (`/ws/{instance_id}`)**：维护连接池字典，确保按 `instance_id` 精准广播。
* **指令转发器 (`/api/{instance_id}/control`)**：接收前端指令，组装 HTTP POST 请求发给对应的 LA IP 和端口。

### 3.2 Web 前端增强设计 (要求现代化工业仪表盘风格)
系统需要两个核心页面：
**页面 1：全局实例管理门户 (`/` 根路由)**
* 展示当前所有已创建的实例卡片，点击卡片进入对应的实例控制台。
* 提供一个“新建实例”表单（输入实例ID、名称、关联的 LA 服务 IP 和端口）。

**页面 2：实例独立控制台 (`/{instance_id}/dashboard`)**
请采用响应式网格布局 (Grid/Flex)，包含以下模块：
1. **顶栏 (Header)**：显示当前实例名称、**WebSocket 连接状态指示灯**（绿点/红点）、**全局静音/取消静音按钮**。
2. **动态控制面板区**：读取本实例配置生成按钮。点击时需有 Loading 状态，接口返回后弹出轻量级的 Toast 成功/失败提示。支持页面端添加新按钮配置。
3. **实时指标展示区**：利用大号字体卡片展示关键字段（如从 Webhook 解析出的“当前节拍”、“生产总数”等，提取 JSON 顶层字段显示）。
4. **日志终端区**：黑底绿字风格的滚动终端。包含**过滤 Tabs**（如“全部”、“预警”、“操作日志”），自动滚动到底部。
5. **智能音频播报模块**：后台隐式运行，匹配 `keyword` 播放预警声音。
