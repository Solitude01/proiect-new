# 回放下载 / 截图 —— 标准示例

> **只需提供：超脑名称 + 通道号 + 时间戳，即可下载 MP4 或抓取截图。**

---

## 快速开始

### 1. 安装依赖

```bash
pip install requests

# 历史截图模式可选（抽帧用）：
# 安装 ffmpeg 并加入 PATH，或: winget install ffmpeg
```

### 2. 三种模式快速上手

```bash
# 模式 A：下载 MP4 视频
python 简易回放下载.py -d 20 -c 5 \
  -b "2026-05-12 10:00:00" -e "2026-05-12 10:05:00"

# 模式 B：实时抓拍（当前画面 → JPEG）
python 简易回放下载.py -d 20 -c 5 --snapshot

# 模式 C：历史时刻截图（回放中抽取一帧）
python 简易回放下载.py -d 20 -c 5 --frame "2026-05-12 10:00:00"
```

---

## 参数说明

| 参数 | 必填 | 说明 | 示例 |
|------|:--:|------|------|
| `-d, --deepmind` | ✅ | 超脑编号，对应 `Deepmind.json` 中 `Deepmind` 字段 | `20`、`5` |
| `-c, --channel` | ✅ | 通道号 | `5`（通道5 → tracks 后缀自动拼为 `501`） |
| `-b, --begin` | 条件 | 开始时间（下载模式） | `"2026-05-12 10:00:00"` |
| `-e, --end` | 条件 | 结束时间（下载模式） | `"2026-05-12 10:05:00"` |
| `--segments` | 条件 | 分段文件（与 -b/-e 二选一） | `segments.txt` |
| `--snapshot` | — | **实时抓拍模式**：抓取当前实时画面 | 无需时间参数 |
| `--frame` | — | **历史截图模式**：`"时间"` 或 `@文件` | `"2026-05-12 10:00:00"` |
| `-o, --out-dir` | 否 | 输出目录（默认脚本所在目录） | `D:\videos` |
| `--tracks` | 否 | 码流后缀：`01`=主码流, `02`=子码流 | `01`（默认） |
| `--timeout` | 否 | 超时秒数（下载默认 300s，截图默认 60s） | `600` |
| `--json` | 否 | 设备配置文件路径 | `Deepmind.json`（默认） |

> **三种模式选其一：** 下载 (`-b -e` / `--segments`) · 实时抓拍 (`--snapshot`) · 历史截图 (`--frame`)

## 批量下载（多段时间）

创建分段文件 `segments.txt`，每行一段（`开始时间 结束时间`）：

```
# segments.txt —— 每行：开始时间 结束时间
2026-05-12 10:00:00  2026-05-12 10:05:00
2026-05-12 11:30:00  2026-05-12 11:32:00
2026-05-12 14:00:00  2026-05-12 14:03:30
```

```bash
python 简易回放下载.py -d 20 -c 5 --segments segments.txt
```

输出文件名格式：`ch{通道}_{海康开始时间}_{海康结束时间}.mp4`

## 📸 截图功能（两种模式）

### 模式 B：实时抓拍 `--snapshot`

抓取摄像头**当前实时画面**，直接保存为 JPEG。无需时间参数。

```bash
python 简易回放下载.py -d 20 -c 5 --snapshot
```

**输出：** `snapshot_ch5_20260512T143025Z.jpg`

**原理：** `GET /ISAPI/Streaming/channels/501/picture` → 返回 JPEG 二进制

---

### 模式 C：历史时刻截图 `--frame`

从**回放录像**中抽取指定时刻的一帧图片。

```bash
# 单时刻截图
python 简易回放下载.py -d 20 -c 5 --frame "2026-05-12 10:00:00"
```

**工作流程：**
1. 自动下载该时刻前后各 1 秒的短视频（共 2 秒）
2. 用 ffmpeg 抽取目标帧 → JPG
3. 自动清理临时短视频

```bash
# 批量历史截图（从文件读取时间点列表）
python 简易回放下载.py -d 20 -c 5 --frame @moments.txt
```

**`moments.txt` 格式**（每行一个时刻）：
```
# 缺陷发生的时刻
2026-05-12 10:00:05
2026-05-12 11:30:22
2026-05-12 14:05:48
```

**输出：**
- 有 ffmpeg：`frame_ch5_20260512T100005Z.jpg`
- 无 ffmpeg：保留短视频 `frame_ch5_20260512T100005Z.mp4`，提示手动抽帧命令

## 码流切换

| 分辨率 | tracks 后缀 | 说明 | 典型场景 |
|--------|:----------:|------|----------|
| **1K（主码流）** | `01` | 高清，文件较大 | 常规回放取证 |
| **2K（子码流）** | `02` | 标清，文件较小 | 快速预览 |

```bash
# 下载子码流（文件更小，下载更快）
python 简易回放下载.py -d 20 -c 5 \
  -b "2026-05-12 10:00:00" -e "2026-05-12 10:05:00" \
  --tracks 02
```

## 设备配置

脚本自动读取同目录下的 `Deepmind.json`，该文件包含所有超脑设备的 IP、密码、端口信息：

```json
{
  "Deepmind": "20",
  "IP": "10.20.5.112",
  "Password": "wdzn6688",
  "HttpPort": 80,
  "RtspPort": 554,
  "Scheme": "http"
}
```

可用编号查询：
```bash
python 简易回放下载.py -d 999 -c 1 -b "2026-01-01 00:00:00" -e "2026-01-01 00:01:00"
# → 会列出所有可用编号
```

## 底层原理

```
输入参数
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 1. 加载 Deepmind.json → 获取设备 IP / 密码 / 端口        │
│ 2. 时间戳转换: "2026-05-12 10:00:00" → "20260512T100000Z" │
│ 3. 构建 playbackURI（内部 & 编码为 %26）                  │
│    rtsp://IP:554/Streaming/tracks/501                    │
│    ?starttime=20260512T100000Z%26endtime=20260512T100500Z│
│ 4. GET /ISAPI/ContentMgmt/download?playbackURI=...       │
│    + HTTP Digest Auth                                    │
│ 5. 流式下载 → .part → os.replace → .mp4                  │
└─────────────────────────────────────────────────────────┘
```

关键技术点：
- **HTTP Digest Auth**：海康 NVR 标准认证方式
- **`trust_env = False`**：禁用系统 HTTP_PROXY 代理，避免内网请求被劫持
- **`.part` 原子化写入**：防止下载中断产生损坏文件
- **`%26` 编码**：playbackURI 作为外层 query 参数，内部 `&` 必须编码

## 常见问题

### 下载相关
- **下载失败 "Invalid Operation"？** 检查时间范围是否有效、通道是否存在。NVR 中该时间段需确有录像。
- **下载很慢或超时？** 增加超时 `--timeout 600`，或切换到子码流 `--tracks 02`。
- **找不到设备？** 确认 `Deepmind.json` 中存在该编号，或用 `--json` 指定路径。
- **能同时下载多个通道吗？** 一次处理一个通道。多通道可多次运行，或用 GUI 版 `batch_download_gui.py`。

### 截图相关
- **历史截图提示 "未安装 ffmpeg"？** 安装 ffmpeg 即可自动抽帧：`winget install ffmpeg`（Windows）或 `apt install ffmpeg`（Linux）。未安装时会保留短视频，可手动抽帧。
- **历史截图为什么不是精确到秒？** 回放视频的最小下载粒度约 2 秒，ffmpeg 从短视频中抽取第 1 秒处的帧（即目标时刻）。精度取决于 NVR 录像的关键帧间隔。
- **实时抓拍和历史截图有什么区别？** `--snapshot` 抓**当前实时**画面；`--frame` 从**历史回放**中截取指定时刻。比如发生缺陷是上午 10 点，现在下午 2 点才来查 → 用 `--frame`。
