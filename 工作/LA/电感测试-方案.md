这里为您总结最终确定的 SOP 检测需求与 LogicAgent 实现逻辑方案。此方案引入了**“状态锁”**机制，解决了“相机持续触发导致状态死循环”的问题。

### 一、 核心需求总结 (SOP Requirements)

1. **硬性顺序**：必须遵循 `空台面校验电感笔`  `放板作业`  `移走板件` 的死循环。
2. **准入条件**：只有在“无板”状态下出现的红灯（电感笔信号）才有效。
3. **防呆控制**：
* 如果在“有板”时校验，无效或报警。
* 如果在“未校验”时直接放板，必须报警。
* 板件一旦移走，之前的校验状态必须清零（强制下次必须重测）。



---

### 二、 LogicAgent 变量定义

请在“变量管理”中预设以下两个全局整型变量：

| 变量名 | 初始值 | 含义 | 作用 |
| --- | --- | --- | --- |
| **`g_IsVerified`** | 0 | 校验通行证 | `0`: 未校验 / 已失效<br>

<br>`1`: 已校验 (持有入场券) |
| **`g_AreaStatus`** | 0 | 区域状态锁 | `0`: 系统认为当前无板<br>

<br>`1`: 系统认为当前有板 |

---

### 三、 完整实现逻辑流程 (The Implementation Flow)

我们将逻辑拆分为三个并行监听的事件流，通过变量进行状态握手。

#### 流程 1：校验电感笔（发券逻辑）

> **目的**：在台面清空时，识别红灯并授予通行证。

1. **[事件匹配]**：检测到 `红灯 (RedLight)` 信号。
2. **[数值比较]**：判断 `g_AreaStatus == 0` ？（检查当前是否为空台面）
* **YES (合规)**  **[变量赋值]** `g_IsVerified = 1` （发放通行证）。
* **NO (违规)**  不动作，或触发语音提示“请先移走板件”。



#### 流程 2：移走板件（复位逻辑 - **含关键锁**）

> **目的**：当板件被拿走时，销毁旧通行证，并解锁区域状态。

1. **[事件匹配]**：检测到 `无板 (NoBoard)` 信号。
2. **[数值比较]**：判断 **`g_AreaStatus == 1`** ？（**关键防抖锁**：确认系统之前是“有板”状态，代表这是“刚刚拿走”的动作）
* **YES (确实拿走了)**  执行复位：
* **[变量赋值]** `g_AreaStatus = 0` （解锁区域）。
* **[变量赋值]** `g_IsVerified = 0` （销毁通行证，防止复用）。


* **NO (一直是空的)**  **什么都不做**（防止相机持续发无板信号导致死循环）。



#### 流程 3：放入板件（核销与生产逻辑）

> **目的**：当板件放入时，检查是否有通行证，并锁定区域。

1. **[事件匹配]**：检测到 `有板 (HasBoard)` 信号。
2. **[数值比较]**：判断 `g_AreaStatus == 0` ？（**防抖**：确保这是新放入的动作，而不是持续的有板信号）
* **YES (新板放入)**  进入核心校验：
* **[数值比较]**：判断 `g_IsVerified == 1` ？（有通行证吗？）
* **YES (有券/OK)** 
1. **[变量赋值]** `g_IsVerified = 0` （核销通行证）。
2. **[变量赋值]** `g_AreaStatus = 1` （**锁定区域**）。
3. **[输出]** 界面显示 OK / 允许下一步测试。


* **NO (无券/NG)** 
1. **[报警控件]** 触发报警 / NG 信号。
2. **[变量赋值] `g_AreaStatus = 1**` （**重要闭环**：即使是违规板，它也占用了台面，必须锁定。否则拿走违规板时无法触发流程2的复位）。

### 四、 方案亮点与逻辑闭环图解

此方案通过 `g_AreaStatus` 实现了**“上升沿/下降沿”**的逻辑效果，避免了工业相机持续信号带来的干扰。

```mermaid
graph TD
    A[初始状态: 无板/无券] -->|流程1: 触发红灯| B{当前是空台面吗?}
    B -- 是 (g_AreaStatus==0) --> C[g_IsVerified = 1 (已发券)]
    B -- 否 --> A

    C -->|流程3: 放入板件| D{有入场券吗?}
    D -- 有 (g_IsVerified==1) --> E[作业OK: 锁台面 g_AreaStatus=1, 销券 g_IsVerified=0]
    D -- 无 --> F[报警NG: 锁台面 g_AreaStatus=1]

    E -->|流程2: 拿走板件| G{刚才是有板状态吗?}
    F -->|流程2: 拿走板件| G
    G -- 是 (g_AreaStatus==1) --> H[系统复位: 解锁 g_AreaStatus=0, 清券 g_IsVerified=0]
    H --> A

```

您可以直接按照这个逻辑图在 LogicAgent 中进行连线配置。
