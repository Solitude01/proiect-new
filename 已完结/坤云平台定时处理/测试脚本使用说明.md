# 坤云平台定时任务测试脚本使用说明

## 问题现象

定时任务**自动执行失败**，但**手动执行**和**立即执行一次**正常。

## 根本原因

`dist/install.ps1` 存在一个关键 bug：**XML 中的变量没有正确展开**，导致计划任务实际执行的命令包含字面量 `$launcherPath` 而不是实际路径。

### Bug 示例（已修复）
```powershell
# 错误的写法（变量名作为字面量）
<Arguments>$taskArgument</Arguments>

# 正确的写法（变量正确展开）
<Arguments>-ExecutionPolicy Bypass -NoProfile -Command "&amp; '$launcherPath'"</Arguments>
```

## 测试脚本

### 1. test_scheduled_task_fix.ps1 - 诊断脚本

这个脚本是专门用来诊断定时任务自动执行失败问题的。

#### 功能
- **诊断现有任务**：分析当前安装的任务配置
- **创建测试任务**：创建多种不同配置的测试任务
- **对比测试**：比较不同执行方式的差异

#### 使用方法

```powershell
# 以管理员身份运行 PowerShell，然后执行：

# 1. 仅诊断现有任务（不创建测试任务）
.\test_scheduled_task_fix.ps1 -DiagnoseOnly

# 2. 完整诊断（创建测试任务并分析）
.\test_scheduled_task_fix.ps1

# 3. 清理所有测试任务
.\test_scheduled_task_fix.ps1 -Cleanup
```

#### 测试方法说明

脚本会创建以下测试任务：

| 任务名 | 说明 |
|--------|------|
| `KY_Diag_FileMode_Bug` | 复现 dist/install.ps1 的 bug（变量未展开） |
| `KY_Diag_FileMode_Correct` | 正确的 -File 模式写法 |
| `KY_Diag_CommandMode` | 推荐的 -Command 模式（兼容性最好） |
| `KY_Diag_FullLauncher` | 模拟产品的完整启动脚本方式 |
| `KY_Diag_ImmediateRun` | 立即执行对比测试 |

### 2. test_scheduledtask_methods.ps1 - 兼容性测试

这个脚本测试多种创建计划任务的方法，用于找出在当前系统上能正常工作的方案。

#### 使用方法

```powershell
# 创建测试任务（每5分钟执行一次）
.\test_scheduledtask_methods.ps1

# 清理测试任务
.\test_scheduledtask_methods.ps1 -Cleanup
```

## 修复步骤

### 步骤1：卸载现有任务

```powershell
# 使用 GUI 工具
# 或运行
.\install.ps1 -Uninstall
```

### 步骤2：验证修复后的安装脚本

两个版本的 `install.ps1` 都已修复：
- `install.ps1`（根目录）- 使用 schtasks.exe 导入 XML
- `dist/install.ps1` - 使用 Register-ScheduledTask -Xml

### 步骤3：重新安装任务

```powershell
# 方法1：使用 PowerShell 脚本（推荐）
.\install.ps1 -IntervalHours 2

# 方法2：使用 GUI 工具
运行 "坤云清理管理工具.exe" → "计划任务" 标签页 → "安装计划任务"
```

### 步骤4：验证安装

```powershell
# 查看任务详情
schtasks /query /tn "坤云平台定时清理" /v /fo list

# 查看任务 XML（重点检查 Arguments 是否正确）
Export-ScheduledTask -TaskName "坤云平台定时清理"

# 手动触发测试
Start-ScheduledTask -TaskName "坤云平台定时清理"

# 查看执行历史
Get-ScheduledTaskInfo -TaskName "坤云平台定时清理"
```

### 步骤5：等待自动执行验证

安装后等待设定的间隔时间（默认2小时），然后检查：
- 日志文件是否生成：`C:\坤云平台定时处理\logs\`
- 任务上次结果是否为 0（成功）

## 关键检查点

### 检查1：启动脚本是否正确生成

```powershell
Get-Content C:\ky_cleanup_launcher.ps1
```

应该显示：
```powershell
# 坤云平台定时清理启动脚本（由安装程序自动生成）
& 'D:\proiect\已完结\坤云平台定时处理\cleanup.ps1' -ConfigPath 'D:\proiect\已完结\坤云平台定时处理\config.json'
```

### 检查2：任务 Arguments 是否正确

```powershell
# 导出任务 XML
$xml = [xml](Export-ScheduledTask -TaskName "坤云平台定时清理")
$xml.Task.Actions.Exec.Arguments
```

应该显示实际路径，而不是 `$launcherPath` 或 `$taskArgument`。

### 检查3：手动执行 vs 自动执行对比

```powershell
# 手动执行（当前用户上下文）
& "C:\ky_cleanup_launcher.ps1"

# 查看日志
Get-ChildItem C:\坤云平台定时处理\logs\ | Sort-Object LastWriteTime -Descending | Select-Object -First 5
```

## 常见问题

### Q: 为什么立即执行成功但定时执行失败？

A: 立即执行和定时执行的主要区别：
1. **用户上下文**：立即执行使用当前用户，定时执行使用 SYSTEM
2. **工作目录**：立即执行在当前目录，定时执行在系统目录
3. **环境变量**：SYSTEM 账户的环境变量不同

### Q: 为什么中文路径会导致问题？

A: SYSTEM 账户的代码页/编码环境与交互式用户不同，中文路径经过 Task Scheduler → powershell.exe 传递时可能编码损坏。解决方案是使用 `-Command` 模式或将脚本放在 ASCII 路径（如 `C:\ky_cleanup_launcher.ps1`）。

### Q: 如何确认问题已修复？

A: 按照以下步骤验证：
1. 重新安装任务
2. 手动执行一次，确认成功
3. 等待一个执行周期（如2小时）
4. 检查是否自动生成新的日志文件
5. 检查任务上次结果是否为 0
