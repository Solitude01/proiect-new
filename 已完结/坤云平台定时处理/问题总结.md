🔧 新问题与修复记录

问题5：schtasks.exe 引号转义错误
- 症状：运行 fix_task_now.ps1 时报错 `错误: 无效参数/选项 - ''C:\ky_cleanup_launcher.ps1''`
- 原因：PowerShell 向 schtasks.exe 传递参数时，引号嵌套导致解析错误
- 修复：使用单引号包裹整个 /tr 参数值，内部使用双单引号转义

问题6：Windows Server 2012 R2 严重兼容性问题（最终根因）
- 症状：所有 PowerShell 方式创建任务均失败，包括：
  - `Register-ScheduledTask` cmdlet 报错（中文名称编码问题）
  - `schtasks.exe` 命令行报错（引号/路径解析失败）
  - PowerShell 脚本创建的 launcher 脚本存在语法错误
  - 任务显示成功执行但无实际日志输出
- 原因：
  - Windows Server 2012 R2 的 PowerShell 4.0 与新版 Windows 存在严重兼容性差异
  - `schtasks.exe` 命令解析器存在 Bug，无法正确处理引号嵌套和中文路径
  - SYSTEM 账户在某些情况下权限受限
  - PowerShell 4.0 的字符串插值导致 `$_` 被错误解析为 GUI 事件对象
- 最终解决方案：
  - 使用 **纯 CMD 批处理** 创建任务（而非 PowerShell）
  - 使用 **英文任务名称** `KY`（避免中文编码问题）
  - 使用 **CMD 包装器** `run_ky.cmd` 调用 PowerShell 脚本
  - 使用最简化的 `schtasks.exe` 命令格式：
    ```cmd
    schtasks.exe /create /tn "KY" /tr "D:\dist\run_ky.cmd" /sc hourly /mo 2 /ru SYSTEM /f
    ```
