Labelme to COCO 转换器 (多文件夹/GUI 增强版)

这是一个功能强大的图形化界面工具，用于将 Labelme 格式的标注数据（JSON + 图片）转换为标准 COCO 数据集格式。它专为处理大规模、多源数据集而设计，支持多文件夹合并、自动数据集切分（训练/测试/验证）、数据质量检查以及统一标签管理。
✨ 主要特性

    现代化 GUI 界面：采用 Material Design 风格设计的图形界面，操作直观流畅。

    多文件夹支持：支持同时导入多个独立的图片文件夹，工具会自动合并处理，无需手动移动文件。

    智能标签管理：

        统一映射：自动扫描所有文件夹，建立全局统一的 Label ID 映射，避免不同文件夹间 ID 冲突。

        标签编辑：支持在界面中直接重命名标签、修改标签 ID、删除多余标签。

        高级替换：支持正则表达式、前缀/后缀批量添加或移除、大小写转换等高级标签清洗功能。

    数据质量检查 (Data QA)：

        自动扫描缺图、缺 JSON、文件损坏、空标注、标注越界、无效多边形/矩形、面积为 0 等 9 类常见错误。

        提供一键修复功能（支持移入回收站或永久删除）。

    自动数据集切分：

        支持自定义训练集 (Train)、测试集 (Test)、验证集 (Verify) 的比例。

        支持设置随机种子 (Random Seed) 以保证切分结果可复现。

    大文件夹优化：

        自动分割：针对包含大量图片的文件夹，支持按数量上限（如 2000 张）自动拆分为多个子文件夹（Part1, Part2...），避免单目录文件过多导致的性能问题。

🛠️ 安装依赖

确保你的 Python 环境版本 >= 3.6。

本工具依赖以下 Python 库，请使用 pip 安装：
Bash

pip install numpy pillow tqdm send2trash

注意：tkinter 通常包含在 Python 标准库中。如果在 Linux 环境下报错，可能需要单独安装 python3-tk。
🚀 快速开始

    启动程序： 在终端中运行脚本：
    Bash

    python label2coco2.7.py

    添加数据：

        点击 "添加文件夹" 或 "添加多个文件夹"。

        选择包含图片 (.jpg, .png 等) 和 Labelme 标注文件 (.json) 的目录。

    标签管理 (可选但推荐)：

        切换到 "标签映射" 选项卡。

        点击 "刷新映射" 查看所有检测到的标签。

        如有需要，可修改标签 ID 或名称，确保数据一致性。

    数据质量检查 (推荐)：

        切换到 "数据质量检查" 选项卡。

        点击 "开始扫描"。

        查看扫描结果，如有错误文件，可使用 "一键修复" 清理脏数据。

    配置与转换：

        在左侧面板设置 输出目录。

        调整 数据集切分比例 (例如：训练 0.8 / 测试 0.1 / 验证 0.1)。

        设置 每文件夹图片上限 (防止单文件夹过大，默认 2000)。

        点击 "开始转换与切分"。

📂 输出目录结构

转换完成后，输出目录将包含以下结构：
Plaintext

Output_Directory/
├── train/
│   ├── images/              # 训练集图片
│   └── annotations/
│       └── instance_train.json  # COCO 格式标注文件
├── test/
│   ├── images/              # 测试集图片
│   └── annotations/
│       └── instance_test.json
├── verify/
│   ├── images/              # 验证集图片
│   └── annotations/
│       └── instance_verify.json
├── label_mapping.txt        # 记录了 Label ID 和名称的对应关系
├── folder_split_info.txt    # 记录了文件夹的切分详情
└── subset_split_info.txt    # 记录了子集的分割详情

注意：如果启用了自动分割功能，images 目录下可能会出现 images_part01, images_part02 等子文件夹结构。
🔍 质量检查项说明

工具会自动检查以下问题：

    缺图片：JSON 文件引用了图片，但文件系统中找不到该图片。

    缺 JSON：存在图片文件，但没有对应的 JSON 标注文件。

    JSON 损坏：JSON 文件格式错误或无法解析。

    空标注：JSON 文件中 shapes 列表为空。

    标注越界：标注点坐标超出了图片宽或高的范围。

    无效多边形：多边形点数少于 3 个。

    无效矩形：矩形标注点数不等于 2 个。

    空标签名：标注的 label 字段为空。

    面积为 0：标注区域计算面积为 0。

📝 注意事项

    备份：虽然工具提供了安全删除（移至回收站）功能，但在进行大规模“一键修复”或“标签批量修改”前，建议备份原始数据。

    随机种子：为了保证每次切分的结果一致（即同样的图片总是分到同一个集），请在配置面板中固定 随机种子 (Random Seed) 的值（默认为 42）。

    编码：工具内部处理了多种字符编码（utf-8, gbk 等），以兼容不同系统生成的 Labelme JSON 文件。
